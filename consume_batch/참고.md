1. 서비스가 먼저 실행 중이어야 함 (이미 된 상태라고 가정)


./target/release/consume_batch
# 또는 systemd 서비스로 등록된 경우
systemctl start consume_batch
2. 같은 바이너리에 --cli 플래그를 붙여서 실행


./target/release/consume_batch --cli
개발 중이라면:


cargo run -- --cli
소켓 경로
소켓 경로는 .env의 SOCKET_PATH 환경변수로 설정 (app_config.rs:87):


SOCKET_PATH=/tmp/consume_batch.sock  # 기본값
